# 最佳实践

QuickNote 高效使用指南。

## 📝 笔记技巧

### 快速记录
- 使用 `⌘K` 随时记录
- 不追求完美，先记录再整理
- 使用 Markdown 格式化

### 结构化笔记
```markdown
# 主题
- 要点1
- 要点2
- [ ] 待办事项
```

### 分类管理
- 使用标签：`#工作 #学习`
- 添加元数据分类
- 定期归档旧笔记

## 🤖 AI 使用

### 模型选择
- 日常：gpt-3.5-turbo（快速便宜）
- 重要：gpt-4（质量最高）
- 长文：claude-3-sonnet（上下文大）

### Prompt 优化
- 明确要求
- 提供示例
- 指定格式

### 定时设置
- 工作日：晚上 23:00
- 周末：可以更晚
- 确保有足够笔记量

## 🌐 Webhook 集成

### 常见场景
1. **Notion 同步**：保存到数据库
2. **Slack 通知**：团队分享
3. **GitHub Issues**：任务管理

### 配置建议
- 2-5 个 Webhook
- 明确用途描述
- 使用 AI 智能路由

## 💡 工作流建议

### 日常流程
1. 随时用 ⌘K 记录
2. 晚上自动生成总结
3. 查看通知确认结果
4. 定期回顾总结

### 周期回顾
- 每周：查看统计数据
- 每月：优化 Prompt
- 每季：调整工作流

## 🔧 性能优化

- 定期清理旧笔记
- 使用本地存储路径
- 选择合适的 AI 模型
- 控制 Webhook 数量
